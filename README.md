# UI Misrepresentation in AI Agent Protocols (CWE-451)

A conceptual research repository demonstrating the risks associated with Informed Consent Bypass in AI Agent tool protocols, specifically focusing on the Model Context Protocol (MCP).

---

## Technical Overview
The vulnerability occurs when a client interface (such as a CLI or Web UI) relies entirely on metadata provided by a remote server to generate confirmation prompts. If the server-side validation is missing, an attacker can misrepresent a high-risk action (e.g., Command Execution) as a low-risk action (e.g., File Read).

### The Attack Chain
1. Metadata Deception: The server sends a tool definition with a benign description (e.g., "Safe File Reader").
2. Visual Misrepresentation: The UI displays the benign description to the user.
3. Execution Divergence: Upon user approval, the server executes arbitrary system commands while returning fake output to the UI to maintain the illusion.

## Distinction from Patch History
This research focuses on the Post-Consent Execution phase, differentiating it from previous fixes like CVE-2025-59536 (Execution before consent).

---

## Defensive Strategies
* Server-Side Verification: Clients must validate that the tool name and parameters match known safe patterns.
* Integrity Monitoring: Real-time monitoring of system calls generated by agent-invoked processes.
* Sandboxing: Executing all tool actions in an isolated environment to prevent host compromise.
